    - Add an HTTP Request node BEFORE the Claude call (after the existing stats aggregation,
      before "Code - Build Prompt") that queries the last 7 daily recaps from Supabase REST:
      `GET {supabaseUrl}/rest/v1/daily_recaps?select=total_calls,recap_date&order=recap_date.desc&limit=7`

    must instruct Claude (haiku) to act as an operations manager writing a morning briefing
    for a regional pest control office, and to output ONLY these sections:

      1. Call volume overview — one line with the WoW comparison, e.g.
         "38 calls today vs. 44-call 7-day average (down 14%)."
      2. Category breakdown — 2-4 bullets naming the dominant call types and their share.
      3. Patterns / flags — bullets ONLY when the day's data clearly supports a pattern
         (e.g. a spike in a specific pest type or a jump in billing calls). If nothing stands
         out, omit this section entirely — do not invent trends.

         cancellations, unresolved callbacks). If none, say "No action items today."

    morning report. Plain text suitable for an email body.

    The prompt MUST explicitly instruct Claude to NOT:
      - mention geographic locations or regions ("customers in South Dakota…")
      - use corporate-speak or filler
      - speculate or extrapolate trends from a single day with no supporting evidence
      - repeat raw stat numbers that are already shown in the stats card; instead add
        interpretation (the stats card already lists totals/categories/flags).

    Keep `claude-haiku-4-5-20251001` as the model. Keep the existing JSON/text extraction
    in "Code - Parse Summary" working (the prompt should still return plain narrative text
    that Parse Summary stores as `summary_text` and renders into `html_body`).
