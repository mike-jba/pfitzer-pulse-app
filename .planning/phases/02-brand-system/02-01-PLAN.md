|-------|-------|------|
| Brand Orange | `#f26522` → `oklch(0.65 0.19 43)` | Primary CTAs, active highlights |

<task type="auto">
  <name>Task 2: Load Montserrat and Open Sans fonts via next/font/google</name>
  <files>src/app/layout.tsx, src/app/globals.css</files>
  <action>
Replace the existing Geist font setup with Montserrat (headings) + Open Sans (body).

In `src/app/layout.tsx`:

1. Remove the existing Geist import:
   ```ts
   // Remove: import { Geist } from "next/font/google";
   // Remove: const geist = Geist({ variable: "--font-geist-sans", subsets: ["latin"] });
   ```

2. Add Montserrat and Open Sans:
   ```ts
   import { Montserrat, Open_Sans } from "next/font/google";

   const montserrat = Montserrat({
     variable: "--font-heading",
     subsets: ["latin"],
     weight: ["400", "600", "700"],
   });

   const openSans = Open_Sans({
     variable: "--font-sans",
     subsets: ["latin"],
     weight: ["400", "500", "600"],
   });
   ```

3. Update the `<html>` className to inject both variables:
   ```tsx
   // was: className={`${geist.variable} h-full antialiased`}
   className={`${montserrat.variable} ${openSans.variable} h-full antialiased`}
   ```

In `src/app/globals.css`, the `@theme inline` block already has:
```css
--font-sans: var(--font-sans);
--font-heading: var(--font-heading);
```
This is technically self-referential in CSS, BUT next/font injects the variable on <html>
scope at runtime, which resolves it. The existing mapping works correctly — no change needed
to globals.css for fonts.

Verify the `html { @apply font-sans; }` base rule still applies — it will pick up Open Sans
via the now-correctly-injected --font-sans variable.
  </action>

`npm run build` passes. Grep layout.tsx confirms `Montserrat` and `Open_Sans` imports exist
and `--font-geist-sans` is gone. `npm run dev` — body text renders in Open Sans (visible

  </verify>
  <done>
