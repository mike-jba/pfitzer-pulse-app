<task type="auto">
  <name>Task 2: Fix the Geist font variable wiring</name>
  <files>src/app/layout.tsx, src/app/globals.css</files>
  <action>
The font is currently broken: layout.tsx sets `variable: "--font-geist-sans"` but
globals.css maps `--font-sans: var(--font-sans)` (self-referential, resolves to nothing).

In `src/app/layout.tsx`, change the Geist config variable:
```ts
const geist = Geist({
  variable: "--font-sans",   // was "--font-geist-sans"
  subsets: ["latin"],
});
```
The `className={`${geist.variable} ...`}` on <html> stays as-is — it will now inject
`--font-sans`.

In `src/app/globals.css`, the `@theme inline` line `--font-sans: var(--font-sans)` now
resolves correctly to the injected Geist variable. Leave it. (If a self-reference lint
warning appears, no action needed — next/font defines the variable on <html> scope.)

Verify `--font-heading: var(--font-sans)` and the `html { @apply font-sans }` base rule
still work — they will, since --font-sans is now real.
  </action>
  <verify>
`npm run dev`, open any page in browser, inspect <body> computed font-family — it resolves
to the Geist font stack, not a fallback serif/system default. Grep layout.tsx confirms
`variable: "--font-sans"`.
  </verify>
  <done>
Geist renders across the app via --font-sans. No self-referential dead variable.
  </done>
</task>
