<objective>
Close the unauthenticated-access hole on the audit endpoints (SEC-01) and document the n8n credential exposure risk (SEC-02).

Purpose: `/api/audit/run` and `/api/audit/call-count` currently accept any request with no auth. Naively adding an INGEST_SECRET Bearer check breaks the app, because these routes are called from the browser (new-audit-modal.tsx) and the secret cannot be exposed client-side. The correct fix is to convert the audit operations to Next.js Server Actions (server-to-server, no client secret) and remove the public unauthenticated API surface. SEC-02 is a documentation-only deliverable.
Output: Audit operations run via server actions; the public unauthenticated audit routes are removed or locked down; a security-notes doc records the accepted n8n credential risk.
</objective>
